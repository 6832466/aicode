# 小说推文错别字修正说明

## 适用场景

- 用户说"帮我改错别字""检查一下这篇文的错别字""修一下错字"
- 文本为小说推文、分镜脚本等长篇内容，字数可能3万字以上
- 需求严格：**只改错别字，数字转汉字，不改内容，不加标点，不改格式**

## 核心原则（必须遵守）

1. **只改错别字**：仅替换明确写错的字，如"的地得"混用、同音错字、形近错字等
2. **数字转汉字**：除了年月日格式的日期外，所有阿拉伯数字转为汉字（详见下方"数字转汉字规则"）
3. **绝不修改内容**：不润色、不删减、不增加、不重写任何句子
4. **绝不添加标点**：原文没有的标点不添加，已有的标点不修改、不删除
5. **格式原样保留**：换行、空行、缩进、分隔符等格式完全照搬
6. **输入什么格式，输出什么格式**：TXT进TXT出，MD进MD出，原文的编码和扩展名不变

## 输出要求（必须遵守）

**只输出修改后的文章内容，不要其他任何东西。**

- ❌ 不输出修改清单
- ❌ 不输出修改统计
- ❌ 不输出进度提示
- ❌ 不输出分析说明
- ❌ 不输出任何与原文无关的额外文字
- ✅ 只输出一篇完整的、修改后的文章，别无他物

## 工作流程

### 步骤1: 确认输入文件

1. 向用户确认待处理的文件路径
2. 读取文件内容
3. 统计总字数，告知用户将分几段处理

### 步骤2: 分段策略

由于文本可能超过3万字，AI无法一次性输出完毕，必须分段处理：

1. **分段规则**：每段约2000-3000字，按自然段落分割（在换行符处断开，不切断句子）
2. **段落数计算**：总字数 ÷ 2500 = 需要的段数（向上取整）
3. 向用户说明分几段处理

### 步骤3: 逐段修正

对每一段执行以下操作：

1. 读取该段原文
2. 逐句扫描，修正错别字 + 将阿拉伯数字转为汉字
3. 将修正后的该段文本追加写入输出文件
4. 简短告知用户当前进度（如"第3/12段已完成"），然后继续下一段

### 步骤4: 输出结果

1. 输出完整修正后的文件（保存到原文件同目录，文件名加 `_fixed` 后缀）
   - 例：`小说.txt` → `小说_fixed.txt`
2. 全部完成后告知用户文件已保存，仅此而已，不输出其他内容

## 错别字判定标准

### 明确修正（必须改）

| 类型 | 示例 |
|------|------|
| 同音错字 | 以经→已经、在见→再见、须要→需要 |
| 形近错字 | 日己→自己、拨打→拨打（拨/拨混用） |
| "的地得"混用 | 高兴的跑→高兴地跑、飞的很高→飞得很高 |
| 常见易错词 | 既然→既然（己然→既然）、帐号→账号 |
| 多字/少字（明确笔误） | 了我→了我（多余字删除） |
| 数字转汉字 | 3个人→三个人、第1次→第一次、20岁→二十岁 |

## 数字转汉字规则

### 必须转换（阿拉伯数字 → 汉字）

| 原文 | 修正 | 说明 |
|------|------|------|
| 3个人 | 三个人 | 数量词 |
| 第1次 | 第一次 | 序数词 |
| 20岁 | 二十岁 | 年龄 |
| 100块 | 一百块 | 金额（口语语境） |
| 5年前 | 五年前 | 时间跨度 |
| 3楼 | 三楼 | 楼层 |
| 2小时 | 两小时 | 时长（注意"二"与"两"的用法） |
| 18号 | 十八号 | 日期中的"号" |
| 80多 | 八十多 | 约数 |
| 3万 | 三万 | 大数（数字+万/亿等单位） |

### 不转换（保持阿拉伯数字原样）

| 原文 | 保持原样 | 原因 |
|------|----------|------|
| 2026年5月25日 | 2026年5月25日 | 年月日格式日期 |
| 1999年 | 1999年 | 年份 |
| 2025年3月 | 2025年3月 | 年月 |
| 3.14 | 3.14 | 小数（小说中罕见，但保留） |

### "二"与"两"的用法规则

- **用"二"**：序数词（第二）、号码（二号）、年份中的数词
- **用"两"**：量词前（两个人、两天）、约数（两三个）
- 常规数量根据语境自然选择，优先遵循口语习惯

### 不修正（保持原样）

| 类型 | 说明 |
|------|------|
| 方言/口语表达 | 如"咋""啥""咋整"等，属于角色语言风格 |
| 网络用语 | 如"给力""666"等，属于时代语境 |
| 古风/文学化表达 | 如倒装句、文言色彩用词 |
| 对话中的语气词 | 如"嗯""啊""哦"等重复使用 |
| 不确定的修改 | 任何无法100%确定是错字的情况，一律不改 |

## 分段处理详细规则

```
1. 读取全文 → 按换行符拆成自然段
2. 将自然段按字数合并为2000-3000字的块
3. 合并时必须在换行符处断开，绝不能切断句子中间
4. 每个块独立处理，互不影响
5. 修正后的块按顺序写入输出文件
```

## 输出文件格式

- 保存到原文件同目录，文件名加 `_fixed` 后缀
- 编码与原文一致（默认UTF-8）
- 格式与原文完全一致（换行符类型CRLF/LF照搬）
- 仅错别字被替换、阿拉伯数字转汉字，其余一字不动
- **文件内容只有修改后的文章本身，无任何额外信息**

## 常见问题处理

- **文件过大无法一次读取**：使用脚本分段读取，每次读取一部分
- **编码问题**：优先尝试UTF-8，失败则尝试GBK/GB2312
- **不确定是否错字**：保持原文不改
- **整段无需修改**：直接原样写入

## 注意事项

- 宁可漏改，不可错改。不确定的绝对不改。
- 分段之间不能丢失内容，每段处理完后确认字数对齐。
- 输出文件必须与原文件格式完全一致，包括换行符类型（CRLF/LF）。
- **输出文件只包含修改后的文章，不要清单、统计、说明等任何额外内容。**
